# MeetingMind Component: Alert

## 1. Overview
A static, inline block that calls out important information to the user.

## 2. Design Philosophy
Alerts should be used sparingly so they don't become "banner blindness." They communicate critical state that the user must know before proceeding.

## 3. Problem Statement
Users might not realize their trial has expired, or that a meeting failed to transcribe, unless it is distinctly called out on the page.

## 4. UX Goals
* Catch the user's eye without totally breaking their flow.
* Provide context and a potential resolution path.

## 5. Usage Guidelines
* Placed at the top of a page, or inline above a form.

## 6. When to Use
* System-wide announcements.
* Form-level error summaries.
* Warning about destructive actions.

## 7. When NOT to Use
* For transient success messages (Use `Toast`).
* For field-level validation (Use inline text below the input).

## 8. Component Anatomy
* Container (Bordered block with a soft background).
* Leading Icon.
* Title.
* Description.
* (Optional) Action Button.

## 9. Variants
1. **Default:** Muted gray/blue (Informational).
2. **Destructive:** Red (Errors, failures).
3. **Warning:** Yellow/Amber (Approaching limits, beta features).
4. **Success:** Green (Rarely used statically, usually a Toast).

## 10. Sizes
* Always `w-full` of its parent container.

## 11. States
* Static.

## 12. Layout Rules
* Block element. Flexbox internally to align icon and text.

## 13. Content Guidelines
* Title should be a short summary (e.g., "Transcription Failed").
* Description should provide the *why* and the *how to fix*.

## 14. Icon Rules
* Default: `Info` or `Terminal`.
* Destructive: `AlertCircle`.
* Warning: `AlertTriangle`.

## 15. Color System
* Default: `bg-background text-foreground border`.
* Destructive: `border-destructive/50 text-destructive dark:border-destructive [&>svg]:text-destructive`.

## 16. Typography
* Title: `font-medium leading-none tracking-tight`.
* Description: `text-sm [&_p]:leading-relaxed`.

## 17. Spacing
* Padding `p-4`. Icon margin `mr-4`.

## 18. Motion
* None (appears instantly).

## 19. Accessibility
* Use `role="alert"` so screen readers announce it immediately when it mounts.

## 20. Keyboard Interaction
* N/A.

## 21. Responsive Behavior
* Fluid width. Stack text if screen is extremely narrow, but usually it wraps.

## 22. Dark Mode
* Borders and text adjust to maintain contrast without harsh glowing backgrounds.

## 23. Design Tokens
* `--destructive`.

## 24. API Specification
```tsx
<Alert variant="destructive">
  <AlertCircle className="h-4 w-4" />
  <AlertTitle>Error</AlertTitle>
  <AlertDescription>
    Your session has expired. Please log in again.
  </AlertDescription>
</Alert>
```

## 25. Props Reference
* `variant`: "default" | "destructive".
