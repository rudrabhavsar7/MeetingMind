# MeetingMind Component: AI Summary Block

## 1. Overview
A specialized content block used to display the output generated by the LLM (summaries, action items, or answers to RAG queries).

## 2. Design Philosophy
AI output should be visually distinct from static database content. It should look slightly "magical" but remain highly readable, supporting Markdown rendering.

## 3. Problem Statement
Dumping raw Markdown text from an LLM onto the screen looks unfinished and doesn't communicate that this content is AI-generated (and therefore subject to hallucination).

## 4. UX Goals
* Visually separate AI content from human content.
* Render rich text (Markdown, lists, bolding).
* Provide feedback mechanisms (Thumbs up/down).

## 5. Usage Guidelines
* Use on the Meeting Details page for the top-level Summary.
* Use on the Ask AI page for the answers.

## 6. When to Use
* Whenever rendering text that came directly from the LLM.

## 7. When NOT to Use
* For the raw transcript (that's human/audio data, not LLM reasoning).

## 8. Component Anatomy
* Container (Card with subtle primary-colored border/glow).
* Header (Sparkles Icon + "AI Generated").
* Content (Markdown Renderer).
* Citations (Links to specific transcript timestamps).
* Footer (Copy button, Feedback thumbs).

## 9. Variants
* **Streaming:** Cursor blinks at the end of the text.
* **Complete:** Shows footer actions.

## 10. Sizes
* `w-full`.

## 11. States
* Generating (Streaming).
* Complete.

## 12. Layout Rules
* Block element.

## 13. Content Guidelines
* Must parse and render Markdown safely.

## 14. Icon Rules
* Always include a `Sparkles` icon in the header.

## 15. Color System
* Background: `bg-muted/30`.
* Border: `border-primary/20`.

## 16. Typography
* Uses `prose prose-sm dark:prose-invert` (Tailwind Typography plugin) for beautiful Markdown rendering.

## 17. Spacing
* `p-6`.

## 18. Motion
* Optional: A very slow, subtle CSS gradient animation on the border while `isStreaming` is true.

## 19. Accessibility
* Ensure the streaming updates are announced appropriately, or wait until generation is complete to announce the block to screen readers.

## 20. Keyboard Interaction
* N/A.

## 21. Responsive Behavior
* Fluid.

## 22. Dark Mode
* Tailwind Typography handles dark mode automatically via `dark:prose-invert`.

## 23. Design Tokens
* `--primary`.

## 24. API Specification
```tsx
<Card className="border-primary/20 bg-muted/30">
  <CardHeader className="flex flex-row items-center gap-2 pb-2">
    <Sparkles className="h-4 w-4 text-primary" />
    <span className="text-xs font-semibold text-primary uppercase">AI Summary</span>
  </CardHeader>
  <CardContent className="prose prose-sm dark:prose-invert max-w-none">
    <ReactMarkdown>{content}</ReactMarkdown>
  </CardContent>
  <CardFooter className="flex justify-end gap-2">
    <Button variant="ghost" size="icon"><Copy /></Button>
    <Button variant="ghost" size="icon"><ThumbsUp /></Button>
  </CardFooter>
</Card>
```

## 25. Props Reference
* `summaryVersionId` (UUID), `version` (Number), `content` (Markdown string), `citations` (Array), `isStreaming` (Boolean).

## 26. Events
* `onCopy`, `onCitationClick`, `onFeedback`. Feedback posts the summary version ID to `/meetings/{meeting_id}/ai-feedback` and remains local by default.

## 27. Composition
* Uses Card, Button, ReactMarkdown.

## 28. AI Usage Guidelines
* This *is* the AI output display.

## 29. Error Handling
* If LLM fails to generate, show an ErrorState inside this block.

## 30. Edge Cases
* LLM outputs broken markdown (e.g., unclosed code blocks). ReactMarkdown handles most of this gracefully.

## 31. Performance
* ReactMarkdown can be slow on massive strings. Memoize if necessary.

## 32. Security Considerations
* **CRITICAL:** Use a sanitizer (like `rehype-sanitize`) with `react-markdown` to prevent XSS if the LLM hallucinates an `<img src=x onerror=alert(1)>` tag.

## 33. Analytics Events
* Store Feedback (Thumbs up/down) in `AIOutputFeedback` for local prompt/evaluation review. Do not send it to an external analytics/model provider automatically.

## 34. Storybook Stories
