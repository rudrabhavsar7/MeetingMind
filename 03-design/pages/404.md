# MeetingMind — 404 Not Found (`app/not-found.tsx`)

The 404 page is a crucial fallback. Users may hit it if they click a dead link or attempt to access a meeting ID that has been deleted (or they don't have access to).

## 1. Page Purpose
To catch broken routing and gracefully guide the user back to safety without breaking the application's immersion.

## 2. Layout Structure

* **Layout:** Full-Screen Focus (No sidebar, no topbar).
* **Container:** `max-w-md` centered both vertically and horizontally.
* **Visuals:**
  * A subtle, muted illustration or a large, faded "404" graphic in the background (`text-slate-100/10` in light mode).
* **Content:**
  * **Headline (`H2`):** "Page not found"
  * **Body (`text-muted-foreground`):** "The meeting or page you are looking for doesn't exist, or you don't have permission to view it."
* **Actions:**
  * Primary Button: "Go to Dashboard" (Links to `/dashboard`).
  * Secondary Button: "Back to previous page" (Uses `router.back()`).

## 3. Implementation (Next.js App Router)

Next.js automatically renders the `not-found.tsx` component when a route doesn't match any directory, or when a developer explicitly calls `notFound()` in a Server Component (e.g., when a meeting ID query returns null).

```tsx
import Link from 'next/link'
import { Button } from '@/components/ui/button'
 
export default function NotFound() {
  return (
    <div className="flex h-screen flex-col items-center justify-center space-y-4">
      <h2 className="text-3xl font-bold tracking-tight">Page not found</h2>
      <p className="text-muted-foreground">Could not find requested resource.</p>
      <Link href="/dashboard">
        <Button>Return to Dashboard</Button>
      </Link>
    </div>
  )
}
```
