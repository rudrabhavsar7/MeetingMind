# Workflow: Memory Update

Use this workflow to keep `PROJECT_MEMORY.md` current.

## When To Update

Update memory after:

- implementation folders are scaffolded
- tech stack choices change
- major API/database contracts are finalized
- known documentation conflicts are resolved
- new local agent workflows or skills are added
- roadmap or product scope changes materially

Do not update memory for tiny copy edits or temporary work.

## Steps

1. Read `PROJECT_MEMORY.md`.
2. Inspect changed files or decisions.
3. Update only the affected sections.
4. Keep memory concise, factual, and operational.
5. Preserve warnings about dirty worktree state if still relevant.
